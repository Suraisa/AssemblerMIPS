It's the brain of the stateMachine which sends the stateMachine to the right state and do a process according of this state.

Parameters
readingCharPointer on the char currently read in the file to decrypt.
lexemeQueuePointeur on a QUEUE_DOUBLE of Lexemes it updates it during the process of lexeme decrypting.
readingValuePointeur on a LIST_DOUBLE, to stock values which don't have lexeme for now (this function will use it, use just an empty list).
stateMachinePointeur on the stateMachine. To know in which LEXEME_STATE are the treatment or to add an error.
lineNumberPointer on an unsigned long int represent the line position in the file to decrypt. It will be incrementated during the process.
finishedFileInteger to know if the file is ended or not.
